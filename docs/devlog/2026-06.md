# 06/06

Added fixed point integers to help with the precision issues I was having with floats in the draw functions before.

Added incremental barycentric just for my test renderer now. When I add it back to the draw functions it should hopefully fix the perf issues I was having before. 

I tried moving flat triangles to fixed point numbers, and there were a few pixel gaps. I think it's because of 

```cpp
constexpr Fixed operator*(const Fixed& f) const {  
  const int64_t intermediate = static_cast<int64_t>(value_) * static_cast<int64_t>(f.value_);  
  
  return from_raw(static_cast<int32_t>(intermediate / scale));  
}
```

That will truncate the last few bytes, meeting that a few pixels can actually get lost. Tried a few solutions, but the one that worked the best is exact integer edge functions. See [[../notes/edge-functions|edge-functions]].
