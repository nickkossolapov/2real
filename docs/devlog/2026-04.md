# 07/04

Implemented `draw::line` using DDA line drawing and got myself a ***super deluxe wireframe cube***.

Was wondering between these two:

```cpp
void line(graphics::Context&, int x0, int y0, int x1, int y1, uint32_t color);
void line(graphics::Context&, Vec2i x, Vec2i y, uint32_t color);
```

With the second, I'd need to create `Vec2i` or use generics. `Vec2i` might be preferred, glm
has [ivec2](https://cglm.readthedocs.io/en/latest/ivec2.html) instead of using generics.

For lines both SDL and Raylib use the individual cords, but for rect Raylib uses x, y, w, h separately while SDL has
`SDL_rect`.

# 20/04

Added a class to manage SDL RAII. Tried a few approaches:
- Initialisation in constructor
  Needed to have an `error` member, and check it. Could silently fail if you forget to check the error, and just more state.
- Factory `SdlContxt::create` method
  I can wrap the return type in `std::expected<SdlContext, InitError>`, but for destructor kept on being called and closing the window
- Initialisation in `init` method
  Returns init errors in a move obvious way. Has the downside that the context is only partially constructed until `init` is called, but this also matches how SDL is called anyway

I ended up going with the third option because it felt like the simplest approach for now.
